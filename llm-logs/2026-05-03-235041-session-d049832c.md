### Overview

This PR wires the existing backend `message.deleteMessage` tRPC endpoint to the frontend via a new server action, adds a "Delete Message" entry to the message hover action bar, and handles optimistic removal from local state. The architecture mirrors the existing pin/edit patterns cleanly.

Previous blocking issues (the `api-client.ts` `input ?? null` regression and `canDeleteAny={canPin}` coupling) are confirmed fixed by the author.

---

### What's Good

- **Server action** (`deleteMessage.ts`) is minimal, typed, and matches the `pinMessage.ts` / `editMessage.ts` conventions exactly.
- **`canModerate` extraction** in `HarmonyShell.tsx` is the right refactor — `canPin` (gated on `!isChannelLocked`) is now cleanly derived from `canModerate`, making the two permissions independently controlled.
- **Loading + error state** on the delete button is thorough: spinner text, timed auto-clear of error messages, timer cleanup on unmount.
- **`onDelete` callback threading** through `MessageList → MessageItem → ActionBar` follows the existing prop pattern.
- Purely cosmetic formatting changes (long line wrapping) are fine.

---

### Must Fix (Blocking)

**1. Deleting a pinned message does not refresh the pinned panel**

This is the same issue FardeenI flagged. `handleDeleteMessage` only removes the message from `localMessages`:

```ts
// HarmonyShell.tsx
const handleDeleteMessage = useCallback((messageId: string) => {
  setLocalMessages(prev => prev.filter(m => m.id !== messageId));
}, []);
```

But `PinnedMessagesPanel` refetches only when `refreshKey` changes (or the panel is reopened/channel changes). If a user deletes a pinned message while the pinned panel is open, the panel continues rendering the deleted message's content — effectively leaking soft-deleted content in the UI until the user manually closes/reopens it.

The same gap exists in the SSE `message:deleted` path if that handler also only calls `setLocalMessages`.

**Suggested fix:** Call `setPinsRefreshKey(prev => prev + 1)` inside `handleDeleteMessage` (and in the SSE `message:deleted` handler if it doesn't already do this):

```ts
const handleDeleteMessage = useCallback((messageId: string) => {
  setLocalMessages(prev => prev.filter(m => m.id !== messageId));
  setPinsRefreshKey(prev => prev + 1); // evict from open pinned panel
}, []);
```

---

### Suggestions (Non-blocking)

**2. `window.confirm` for the confirmation dialog**

[`MessageItem.tsx:497`](harmony-frontend/src/components/message/MessageItem.tsx) uses `window.confirm('Delete this message? This cannot be undone.')`. This is a native browser dialog that doesn't match the app's design language, can be suppressed by pop-up blockers, and provides no loading feedback since it's synchronous. A lightweight inline confirmation (e.g., toggling to "Are you sure? [Confirm] [Cancel]" in the dropdown) would be more consistent. This is non-blocking but worth addressing.

**3. Duplicate error handling in `handleDelete`**

The `catch` block in `handleDelete` is identical to the `!result.ok` branch body. Extract a small helper:

```ts
const showDeleteError = (msg: string) => {
  setDeleteErrorMsg(msg);
  setDeleteState('error');
  if (deleteErrorTimerRef.current) clearTimeout(deleteErrorTimerRef.current);
  deleteErrorTimerRef.current = setTimeout(() => {
    setDeleteState('idle');
    setDeleteErrorMsg('');
  }, 3000);
};
```

**4. No automated tests**

The test plan is manual-only. The `deleteMessageAction` server action and the `handleDelete` callback have enough branching that a Jest unit test covering the `ok: true`, `ok: false (forbidden)`, and `throw` paths would be valuable. Non-blocking for this PR but should be tracked.

---

### Verdict

**Changes requested** — the pinned panel leak (item 1) is a real data-visibility bug that should be fixed before merge. Items 2–4 can be addressed in follow-up.

import os, glob, json, subprocess
from datetime import datetime

cwd = os.getcwd()
project_key = cwd.replace('/', '-').replace('.', '-')
project_dir = os.path.expanduser(f'~/.claude/projects/{project_key}')

if not os.path.isdir(project_dir):
    print(f'ERROR: project dir not found: {project_dir}')
    exit(1)

files = glob.glob(os.path.join(project_dir, '*.jsonl'))
if not files:
    print('ERROR: no session JSONL files found')
    exit(1)

latest = max(files, key=os.path.getmtime)
session_id = os.path.splitext(os.path.basename(latest))[0]
short_id = session_id[:8]

now = datetime.now()
date_str = now.strftime('%Y-%m-%d')
time_str = now.strftime('%H%M%S')
outfile = f'llm-logs/{date_str}-{time_str}-session-{short_id}.md'

os.makedirs('llm-logs', exist_ok=True)

lines = []
with open(latest) as f:
    for line in f:
        line = line.strip()
        if not line:
            continue
        try:
            lines.append(json.loads(line))
        except json.JSONDecodeError:
            continue

md = [f'# Session Log {date_str}-{time_str}\n']

for entry in lines:
    t = entry.get('type')
    if t == 'user':
        msg = entry.get('message', {})
        content = msg.get('content', '')
        if isinstance(content, list):
            texts = [b.get('text', '') for b in content if isinstance(b, dict) and b.get('type') == 'text']
            content = '\n'.join(texts)
        if content.strip():
            md.append(f'## User\n\n{content.strip()}\n')
    elif t == 'assistant':
        msg = entry.get('message', {})
        content = msg.get('content', '')
        if isinstance(content, list):
            parts = []
            for b in content:
                if not isinstance(b, dict):
                    continue
                if b.get('type') == 'text':
                    parts.append(b.get('text', ''))
                elif b.get('type') == 'tool_use':
                    name = b.get('name', 'tool')
                    inp = b.get('input', {})
                    inp_str = json.dumps(inp, indent=2) if inp else ''
                    parts.append(f'**Tool call: `{name}`**\n```json\n{inp_str}\n```')
                elif b.get('type') == 'tool_result':
                    result_content = b.get('content', '')
                    if isinstance(result_content, list):
                        result_content = '\n'.join(
                            r.get('text', '') for r in result_content if isinstance(r, dict)
                        )
                    parts.append(f'**Tool result:**\n```\n{str(result_content)[:2000]}\n```')
            content = '\n\n'.join(p for p in parts if p.strip())
        if content.strip():
            md.append(f'## Assistant\n\n{content.strip()}\n')

with open(outfile, 'w') as f:
    f.write('\n'.join(md) + '\n')

print(f'OUTFILE={outfile}')

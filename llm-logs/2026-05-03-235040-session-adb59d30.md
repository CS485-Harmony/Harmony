## PR #609 Review — [fix: preserve voice connection across text channel switches](https://github.com/CS485-Harmony/Harmony/pull/609)

**Author:** Declan Blanc | **+416 / -250** | Related: Issue #122

---

### Overview

The PR solves a real bug: `VoiceProvider` was living inside `HarmonyShell`, a page-level component that unmounts on every channel navigation, causing the Twilio room to disconnect silently. The fix moves `VoiceProvider` up to a Next.js App Router `[serverSlug]` layout, which stays mounted across child-route navigations. This is the correct architectural move.

---

### What's Good

- **Core architecture is correct.** App Router layouts are exactly the right tool for this — the provider survives text-channel switches and remounts cleanly on server switches.
- **Named SSE handlers** (`handleVoiceUserJoined`, etc.) replacing inline `useCallback` inside the `useServerEvents` call are cleaner and easier to test/reason about.
- **`currentUserId` added to `joinChannel` deps** ([VoiceContext.tsx:891](harmony-frontend/src/contexts/VoiceContext.tsx)) — this is a genuine latent bug fix; the callback used `currentUserId` but it wasn't listed as a dependency.
- **Participant state pruning on channel delete** (the `idSet` filter in the new `useEffect`) prevents stale sidebar counts from lingering after a voice channel is removed.
- **`setVoiceChannelIds` deduplication guard** (`prev.join(',') === nextKey ? prev : channelIds`) correctly avoids triggering downstream effects when the content hasn't actually changed.
- **New test** verifies that `setVoiceChannelIds` is called with the updated IDs after a channel-created SSE event fires.

---

### Issues

#### Medium — Dead Code: `VoiceExternalActions` / `externalActionsRef`

[VoiceContext.tsx:127–141](harmony-frontend/src/contexts/VoiceContext.tsx) — The `VoiceExternalActions` interface and `externalActionsRef` prop on `VoiceProvider` are no longer used by any consumer. `HarmonyShell` dropped its `voiceActionsRef` entirely, and the three methods are now exposed directly on `VoiceContextValue`. The `useEffect` that syncs into `externalActionsRef` (line ~266) is still present. This should be removed to avoid future confusion about which communication path is canonical.

#### Minor — Redundant dependency in `HarmonyShell` effect

```tsx
// HarmonyShell.tsx
const localVoiceChannelIdsKey = localVoiceChannelIds.join(',');
useEffect(() => {
  setVoiceChannelIds?.(localVoiceChannelIds);
}, [setVoiceChannelIds, localVoiceChannelIds, localVoiceChannelIdsKey]); // ← redundant
```

`localVoiceChannelIds` (array from `useMemo`) and `localVoiceChannelIdsKey` (string derived from it) change together — listing both is redundant and slightly misleading. Since `setVoiceChannelIds` already does the string-comparison dedup guard internally, you can drop `localVoiceChannelIdsKey` from deps entirely and rely on the guard in `VoiceContext`.

#### Minor — Double-initialization of `liveVoiceChannelIds`

```tsx
// VoiceContext.tsx
const [liveVoiceChannelIds, setLiveVoiceChannelIds] = useState<string[]>(voiceChannelIds);
// ...
useEffect(() => {
  setLiveVoiceChannelIds(voiceChannelIds); // sets same value on mount
}, [serverId, voiceChannelIds]);
```

On initial mount the state is initialized from props AND the effect runs and sets the same value again (harmless but wasteful). The effect is the right mechanism for resetting on server change; the `useState` initializer is fine for SSR hydration. Not a bug, but worth noting.

#### Minor — Silent failure swallowed in layout

```tsx
// layout.tsx
} catch {
  // Data fetch failed — render children without voice context.
}
```

Swallowing all errors here means the user sees a functional app but with no voice capability and no indication why. At minimum, logging the error (`console.error`) would help in production debugging. Alternatively, narrowing the catch to only network/auth errors and letting unexpected errors surface would be safer.

---

### Test Coverage Gaps

- Manual test items 1 and 2 (join voice + navigate text channels; switch servers) are unchecked in the PR checklist. These are the core acceptance criteria for this fix and should be verified before merge.
- The full backend Jest suite was blocked locally — the PR notes `tests/voice.service.test.ts` passed individually, but full suite verification is pending.
- No test covers the server-switch case (VoiceProvider unmounts correctly on `serverSlug` change).

---

### Summary

The approach is architecturally sound and solves the bug correctly. **Before merging, two things should be addressed:**

1. **Remove `VoiceExternalActions` / `externalActionsRef`** — it's dead code now and will confuse future readers.
2. **Verify the two unchecked manual test items** (voice-through-text-navigation and server-switch disconnect).

The redundant dependency and double-initialization are polish issues that can be cleaned up without blocking the PR.

```python
import os, glob, json, subprocess
from datetime import datetime

cwd = os.getcwd()
project_key = cwd.replace('/', '-').replace('.', '-')
project_dir = os.path.expanduser(f'~/.claude/projects/{project_key}')

if not os.path.isdir(project_dir):
    print(f'ERROR: project dir not found: {project_dir}')
    exit(1)

files = glob.glob(os.path.join(project_dir, '*.jsonl'))
if not files:
    print('ERROR: no session JSONL files found')
    exit(1)

latest = max(files, key=os.path.getmtime)
session_id = os.path.splitext(os.path.basename(latest))[0]
short_id = session_id[:8]

now = datetime.now()
date_str = now.strftime('%Y-%m-%d')
time_str = now.strftime('%H%M%S')
outfile = f'llm-logs/{date_str}-{time_str}-session-{short_id}.md'

os.makedirs('llm-logs', exist_ok=True)

lines = []
with open(latest) as f:
    for line in f:
        line = line.strip()
        if not line:
            continue
        try:
            lines.append(json.loads(line))
        except json.JSONDecodeError:
            continue

md = [f'# Session Log {date_str}-{time_str}\n']

for entry in lines:
    t = entry.get('type')
    if t == 'user':
        msg = entry.get('message', {})
        content = msg.get('content', '')
        if isinstance(content, list):
            texts = [b.get('text', '') for b in content if isinstance(b, dict) and b.get('type') == 'text']
            content = '\n'.join(texts)
        if content.strip():
            md.append(f'## User\n\n{content.strip()}\n')
    elif t == 'assistant':
        msg = entry.get('message', {})
        content = msg.get('content', '')
        if isinstance(content, list):
            parts = []
            for b in content:
                if not isinstance(b, dict):
                    continue
                if b.get('type') == 'text':
                    parts.append(b.get('text', ''))
                elif b.get('type') == 'tool_use':
                    name = b.get('name', 'tool')
                    inp = b.get('input', {})
                    inp_str = json.dumps(inp, indent=2) if inp else ''
                    parts.append(f'**Tool call: `{name}`**\n```json\n{inp_str}\n```')
                elif b.get('type') == 'tool_result':
                    result_content = b.get('content', '')
                    if isinstance(result_content, list):
                        result_content = '\n'.join(
                            r.get('text', '') for r in result_content if isinstance(r, dict)
                        )
                    parts.append(f'**Tool result:**\n```\n{str(result_content)[:2000]}\n```')
            content = '\n\n'.join(p for p in parts if p.strip())
        if content.strip():
            md.append(f'## Assistant\n\n{content.strip()}\n')

with open(outfile, 'w') as f:
    f.write('\n'.join(md) + '\n')

print(f'OUTFILE={outfile}')
```

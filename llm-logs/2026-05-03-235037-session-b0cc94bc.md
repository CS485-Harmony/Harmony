### Overview

This PR fixes a `404`-surfacing bug in the admin SEO preview endpoint when a valid `PUBLIC_INDEXABLE` channel has no persisted `generated_meta_tags` row. The fix adds an ephemeral fallback preview path on the backend and sanitizes the error surface on the frontend so users never see raw server-component error text.

### Issues

**1. Cache key collision risk — `fallbackPreviewCacheKey` vs existing key patterns**

[`admin.metaTag.router.ts`](harmony-backend/src/routes/admin.metaTag.router.ts)

The fallback cache key is:
```
meta-tag:admin-preview-fallback:<channelId>
```
The existing `cacheService` key helpers (e.g. `metaChannel`, `analysisChannel`) all follow the pattern:
```
<domain>:channel:<channelId>
```
The new key inverts this (`meta-tag:...-fallback:<channelId>`) and is defined inline in the router rather than in `cache.service.ts` alongside the other helpers. This is a minor inconsistency — it's unlikely to cause a collision but it diverges from the established convention in [`cache.service.ts:27–32`](harmony-backend/src/services/cache.service.ts). If this is intentional (router-local cache key, not a shared concern), add a short comment explaining why it lives here rather than in `cacheService.keys`.

**2. 60-second TTL is very short for a fallback that triggers an LLM/generation call**

[`admin.metaTag.router.ts:32`](harmony-backend/src/routes/admin.metaTag.router.ts)

```ts
const FALLBACK_PREVIEW_TTL_SECONDS = 60;
```

`getFallbackMetaTagsForPreview` calls `generateMetaTagsFromContext`, which is presumably the same generation pipeline used for normal regeneration jobs. A 60s TTL means a busy admin reloading the preview panel can re-trigger generation every minute. The existing regeneration job cache likely uses a much longer TTL. Consider aligning to a higher value (e.g. 5–10 minutes) or tying it to the existing `REGENERATION_CACHE_TTL` constant if one exists, to avoid repeated expensive generation calls from the admin UI.

**3. Fire-and-forget cache write can silently fail and produce inconsistent cache state**

[`admin.metaTag.router.ts:150–153`](harmony-backend/src/routes/admin.metaTag.router.ts)

```ts
cacheService
  .set(cacheKey, fallbackPreview, { ttl: FALLBACK_PREVIEW_TTL_SECONDS })
  .catch((err) =>
    logger.warn({ err, channelId, cacheKey }, 'Failed to cache admin SEO fallback preview'),
  );
return fallbackPreview;
```

The fire-and-forget pattern is acceptable here (cache miss on failure is fine), but the `catch` only logs a `warn`. If Redis is down and the caller retries multiple times within 60s, each call will regenerate because the cache write never landed. This is a degenerate case, but a warn in the `set` path combined with a warn in the `get` path could result in two warnings per request with no observable spike to ops. This is low severity but worth noting.

**4. `SEO_PREVIEW_LOAD_ERROR` constant is used correctly but the module is very thin**

[`harmony-frontend/src/lib/seoConstants.ts`](harmony-frontend/src/lib/seoConstants.ts)

A single-export file that holds one string is fine for now. No issue here.

**5. The `catch` clause in `SeoPreviewSection.tsx` silently discards the error**

[`SeoPreviewSection.tsx:90`](harmony-frontend/src/components/settings/SeoPreviewSection.tsx)

```ts
} catch {
  if (cancelled) return;
  setError(SEO_PREVIEW_LOAD_ERROR);
}
```

The error is intentionally dropped here because `fetchSeoPreview` (the server action) now re-throws a sanitized `Error(SEO_PREVIEW_LOAD_ERROR)` — so there's nothing useful to log on the client side. This is correct. However, if `fetchSeoPreview` is ever changed to throw a structured error with additional context, that information will be silently swallowed here. A minimal `console.warn(err)` in development would future-proof this, though it's not a blocker.

```python
    exit(1)

files = glob.glob(os.path.join(project_dir, '*.jsonl'))
if not files:
    print('ERROR: no session JSONL files found')
    exit(1)

latest = max(files, key=os.path.getmtime)
session_id = os.path.splitext(os.path.basename(latest))[0]
short_id = session_id[:8]

now = datetime.now()
date_str = now.strftime('%Y-%m-%d')
time_str = now.strftime('%H%M%S')
outfile = f'llm-logs/{date_str}-{time_str}-session-{short_id}.md'

os.makedirs('llm-logs', exist_ok=True)

lines = []
with open(latest) as f:
    for line in f:
        line = line.strip()
        if not line:
            continue
        try:
            lines.append(json.loads(line))
        except json.JSONDecodeError:
            continue

md = [f'# Session Log {date_str}-{time_str}\n']

for entry in lines:
    t = entry.get('type')
    if t == 'user':
        msg = entry.get('message', {})
        content = msg.get('content', '')
        if isinstance(content, list):
            texts = [b.get('text', '') for b in content if isinstance(b, dict) and b.get('type') == 'text']
            content = '\n'.join(texts)
        if content.strip():
            md.append(f'## User\n\n{content.strip()}\n')
    elif t == 'assistant':
        msg = entry.get('message', {})
        content = msg.get('content', '')
        if isinstance(content, list):
            parts = []
            for b in content:
                if not isinstance(b, dict):
                    continue
                if b.get('type') == 'text':
                    parts.append(b.get('text', ''))
                elif b.get('type') == 'tool_use':
                    name = b.get('name', 'tool')
                    inp = b.get('input', {})
                    inp_str = json.dumps(inp, indent=2) if inp else ''
                    parts.append(f'**Tool call: `{name}`**\n```json\n{inp_str}\n```')
                elif b.get('type') == 'tool_result':
                    result_content = b.get('content', '')
                    if isinstance(result_content, list):
                        result_content = '\n'.join(
                            r.get('text', '') for r in result_content if isinstance(r, dict)
                        )
                    parts.append(f'**Tool result:**\n```\n{str(result_content)[:2000]}\n```')
            content = '\n\n'.join(p for p in parts if p.strip())
        if content.strip():
            md.append(f'## Assistant\n\n{content.strip()}\n')

with open(outfile, 'w') as f:
    f.write('\n'.join(md) + '\n')

print(f'OUTFILE={outfile}')
```

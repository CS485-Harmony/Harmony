## Code Review

### Overview

This PR fixes a genuine security gap: SSE streams for `/server/:serverId` and `/channel/:channelId` only validated membership at connection time. A removed user could continue receiving private events. The fix adds two complementary layers: immediate close on `MEMBER_LEFT` event (reactive) + periodic revalidation polling (defensive fallback).

---

### Issues

**1. Race condition: `closeSseForRevokedAccess` calls `cleanupSseConnection` without setting `state.closed`**

[events.router.ts:185](harmony-backend/src/routes/events.router.ts)

```typescript
function closeSseForRevokedAccess(res, state, subscriptions) {
  if (state.closed) return;
  cleanupSseConnection(state, subscriptions);  // does NOT set state.closed
  ...
}
```

`cleanupSseConnection` clears intervals and unsubscribes but never sets `state.closed = true`. After this function returns, a concurrent `MEMBER_LEFT` event or a revalidation tick could re-enter and call `res.end()` again. `state.closed` should be set to `true` at the top of `closeSseForRevokedAccess`, before any cleanup, to make the guard effective:

```typescript
function closeSseForRevokedAccess(res, state, subscriptions) {
  if (state.closed) return;
  state.closed = true;  // guard against re-entry
  cleanupSseConnection(state, subscriptions);
  ...
}
```

**2. `canAccessChannelStream` makes up to 3 sequential DB round-trips**

[events.router.ts:204](harmony-backend/src/routes/events.router.ts)

```typescript
async function canAccessChannelStream(userId, channelId, serverId) {
  const membership = await prisma.serverMember.findFirst(...);   // trip 1
  const channel = await prisma.channel.findUnique(...);           // trip 2
  const channelMembership = await prisma.channelMember.findUnique(...); // trip 3
}
```

This is called during both initial authorization and on every periodic revalidation tick (every 30 s). Trips 1 and 2 could run in parallel since neither depends on the other. More importantly, at the revalidation call site the `channelId` and `serverId` are already known, so the channel lookup in trip 2 (which re-fetches `channel.serverId` only to re-confirm it equals the `serverId` already passed in) is redundant on revalidation. At minimum, trips 1 and 2 can be parallelized:

```typescript
const [membership, channel] = await Promise.all([
  prisma.serverMember.findFirst(...),
  prisma.channel.findUnique(...),
]);
```

**3. Revalidation interval leaks into test suite via `process.env` mutation**

[events.router.test.ts:345](harmony-backend/tests/events.router.test.ts), [events.router.server.test.ts:595](harmony-backend/tests/events.router.server.test.ts)

Both revalidation tests mutate `process.env.SSE_MEMBERSHIP_REVALIDATION_INTERVAL_MS` and clean it up in the happy path, but if the `new Promise` rejects early (e.g., the 1-second timeout fires), `delete process.env.SSE_MEMBERSHIP_REVALIDATION_INTERVAL_MS` is never reached. This can bleed the 20 ms interval into later tests. The cleanup should be in a `finally` block or `afterEach` to be safe:

```typescript
afterEach(() => {
  delete process.env.SSE_MEMBERSHIP_REVALIDATION_INTERVAL_MS;
});
```

### Minor Observations (non-blocking)

- `startAuthorizationRevalidation` logs `'SSE membership revalidation failed; keeping stream open'` on DB errors — this is correct fail-open behavior for a defensive check, but worth a brief comment noting the intentional choice to avoid false disconnects on transient DB errors.
- The `MEMBER_LEFT` subscription on the channel route subscribes once before `finalizeSseSetup`, which is consistent with how the existing subscriptions are structured. No concern here.
- `state.authorizationRevalidation.unref?.()` is a thoughtful touch to avoid keeping the Node process alive in test environments.

---

**Summary:** Issue #1 (missing `state.closed = true`) is a correctness bug and should be fixed before merge. Issue #2 is a performance concern on the revalidation hot path. Issue #3 is a test reliability risk. The overall approach is well-designed — the dual reactive+polling strategy is the right architecture for this problem.

```python
import os, glob, json, subprocess
from datetime import datetime

cwd = os.getcwd()
project_key = cwd.replace('/', '-').replace('.', '-')
project_dir = os.path.expanduser(f'~/.claude/projects/{project_key}')

if not os.path.isdir(project_dir):
    print(f'ERROR: project dir not found: {project_dir}')
    exit(1)

files = glob.glob(os.path.join(project_dir, '*.jsonl'))
if not files:
    print('ERROR: no session JSONL files found')
    exit(1)

latest = max(files, key=os.path.getmtime)
session_id = os.path.splitext(os.path.basename(latest))[0]
short_id = session_id[:8]

now = datetime.now()
date_str = now.strftime('%Y-%m-%d')
time_str = now.strftime('%H%M%S')
outfile = f'llm-logs/{date_str}-{time_str}-session-{short_id}.md'

os.makedirs('llm-logs', exist_ok=True)

lines = []
with open(latest) as f:
    for line in f:
        line = line.strip()
        if not line:
            continue
        try:
            lines.append(json.loads(line))
        except json.JSONDecodeError:
            continue

md = [f'# Session Log {date_str}-{time_str}\n']

for entry in lines:
    t = entry.get('type')
    if t == 'user':
        msg = entry.get('message', {})
        content = msg.get('content', '')
        if isinstance(content, list):
            texts = [b.get('text', '') for b in content if isinstance(b, dict) and b.get('type') == 'text']
            content = '\n'.join(texts)
        if content.strip():
            md.append(f'## User\n\n{content.strip()}\n')
    elif t == 'assistant':
        msg = entry.get('message', {})
        content = msg.get('content', '')
        if isinstance(content, list):
            parts = []
            for b in content:
                if not isinstance(b, dict):
                    continue
                if b.get('type') == 'text':
                    parts.append(b.get('text', ''))
                elif b.get('type') == 'tool_use':
                    name = b.get('name', 'tool')
                    inp = b.get('input', {})
                    inp_str = json.dumps(inp, indent=2) if inp else ''
                    parts.append(f'**Tool call: `{name}`**\n```json\n{inp_str}\n```')
                elif b.get('type') == 'tool_result':
                    result_content = b.get('content', '')
                    if isinstance(result_content, list):
                        result_content = '\n'.join(
                            r.get('text', '') for r in result_content if isinstance(r, dict)
                        )
                    parts.append(f'**Tool result:**\n```\n{str(result_content)[:2000]}\n```')
            content = '\n\n'.join(p for p in parts if p.strip())
        if content.strip():
            md.append(f'## Assistant\n\n{content.strip()}\n')

with open(outfile, 'w') as f:
    f.write('\n'.join(md) + '\n')

print(f'OUTFILE={outfile}')
```
